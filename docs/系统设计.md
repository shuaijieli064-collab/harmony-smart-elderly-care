# 系统设计

## 1. 系统架构设计

### 1.1 整体架构

```
┌─────────────────────────────────────────┐
│           社区管理端 (Web)              │
├─────────────────────────────────────────┤
│  表示层: Web页面                        │
│  业务层: Java Spring Boot              │
│  数据层: MySQL + REST API               │
└────────────────────┬──────────────────┘
                     │ HTTP/REST
        ┌────────────┴────────────┐
        │                         │
┌───────▼───────┐        ┌────────▼────────┐
│   家庭端 APP  │        │   个人端 APP    │
│  (HarmonyOS)  │        │  (HarmonyOS)    │
├───────────────┤        ├─────────────────┤
│  表示层: ArkUI │        │  表示层: ArkUI  │
│  业务层: ArkTS│        │  业务层: ArkTS │
│  数据层: SQLite│       │  数据层: SQLite│
│ + 分布式数据  │        │ + 分布式数据   │
└───────────────┘        └─────────────────┘
```

## 3. 适老化设计规范

### 3.1 交互设计

- 最小触控区域：48dp × 48dp
- 按钮高度：最低56dp
- 文字大小：标题24sp，正文18sp
- 图标尺寸：32dp以上
- 色彩对比度：≥4.5:1

### 3.2 信息密度

- 单屏信息项：≤5项
- 列表每行：1-2个信息点
- 图表数据点：≤7个

### 3.3 反馈机制

- 操作确认：文字+图标双重提示
- 成功反馈：绿色+弹窗
- 错误反馈：红色+具体说明
- 加载状态：进度条+文字说明

## 4. 安全设计

### 4.1 数据加密

- 传输加密：HTTPS
- 存储加密：AES-256

### 4.2 权限控制

基于RBAC的权限模型：
- 管理员：全部权限
- 家庭成员：查看、关怀权限
- 老年用户：录入、自查权限

## 5. 异常检测算法

### 5.1 阈值检测

```
IF value > upper_threshold OR value < lower_threshold THEN
    trigger alert
END IF
```

### 5.2 滑动窗口趋势分析

```
window_size = 7
threshold_change = 20%

计算窗口内平均值变化率
IF change_rate > threshold_change THEN
    trigger warning
END IF
```
