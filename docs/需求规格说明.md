# 需求规格说明

## 1. 系统概述

### 1.1 项目背景

本项目旨在构建一套基于鸿蒙分布式能力的社区智慧医养监护系统，解决当前智慧养老领域存在的系统数据割裂、强依赖硬件设备、适老化设计不足等问题。

### 1.2 系统目标

实现健康数据在社区、家庭、个人三级主体间的安全共享与异常预警。

## 2. 用户角色

| 角色 | 描述 | 权限 |
|------|------|------|
| 社区管理员 | 社区服务中心管理人员 | 社区台账管理、全局数据查看 |
| 家庭成员 | 老年人家属 | 远程关怀、健康数据查看 |
| 老年用户 | 社区老年人 | 健康数据录入、日常提醒 |

## 3. 功能需求

### 3.1 健康数据采集

- 支持手动录入健康数据（血压、心率、体温等）
- 支持查看历史健康记录
- 支持数据趋势展示

### 3.2 智能提醒

- 用药提醒
- 体检提醒
- 运动提醒

### 3.3 异常预警

- 基于阈值设置的超限提醒
- 基于滑动窗口的趋势分析预警
- 多级预警机制（普通、紧急、严重）

### 3.4 社区台账管理

- 社区居民健康档案管理
- 健康数据统计分析
- 批量提醒设置

### 3.5 亲情远程关怀

- 家庭成员远程查看老人健康状态
- 异常状态推送通知
- 远程留言/语音关怀

## 4. 非功能需求

### 4.1 性能需求

- 应用启动时间 < 3秒
- 数据同步延迟 < 2秒
- 异常预警响应时间 < 1秒

### 4.2 安全需求

- 数据传输加密
- 本地数据存储加密
- 基于RBAC的权限控制

### 4.3 适老化需求

- 大按钮设计（最小48dp）
- 低信息密度显示
- 语义化提示文字
- 字体大小可调整

### 4.4 兼容性

- 支持HarmonyOS 4.0及以上版本
- 支持多设备协同（手机、平板）
